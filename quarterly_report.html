How Radar Availability is Calculated?

What is a Found Data Slot?

A data slot represents a 6-minute time period (00, 06, 12, 18, 24, 30, 36, 42, 48, 54 minutes past each hour), creating 10 slots per hour and 240 slots per day. A slot is marked as "found" when the system detects radar data files for that specific time period from any of two sources:

• Source 1 - Raw files

Checks for files with the exact minutes 00, 06, 12, 18, 24, 30, 36, 42, 48, 54 accessed via radarop@rad19-t1. TMS and TCR raw directories are /backup_data/TMS_IrisFormatRaw_archive and /backup_data/TCR_IrisFormatRaw_archive respectively.

• Source 2 - Rapid-scan images

Checks if any rapid-scan image exists within the 6-minute interval. Images of TMS and TCR are "TMS RS > RS PPI > dBZ(H), 64km (TMS_RAPID_SCAN/TMS_RS_PPI_Z_000_064)" and "TCR RS > RS PPI > dBZ(H), 64km (Radar_TCR_rapid_scan/TCR_RS_Z_064km)" respectively.

If EITHER the raw files OR the rapid-scan images contain data for a time slot, that slot counts as "found".

For e.g., raw file of TCR is missing at exactly 16:54 on 2 Jun 2025, but rapid-scan image is found at 16:58, therefore the entire 16:54 slot is marked as available.

TMS / TCR Availability:

Availability = (found data slots that are not in maintenance / expected slots) × 100%. Expected slots = 240 per day × number of days in that month.

For e.g., TMS is under maintenance between 02:34 and 20:27 on 13 Jun 2025. Therefore, data between 02:30 and 20:29 is counted as unavailable.

Preventive Maintenance Adjustment:

When preventive maintenance occurs, those time slots are subtracted from the expected slots, providing a more accurate availability percentage that excludes planned downtime.

Data Restoration:

When a "Data Restoration" record is added, it indicates that data previously thought missing for that period actually exists. The corresponding time slots are added back as found data, overriding any type of maintenance (preventive, corrective, or other). This ensures that availability calculations reflect the true data presence.

AND Logic:

Shows availability when both stations must have data simultaneously. A time slot is considered available only if both TMS and TCR have data and neither is in maintenance.

OR Logic:

Shows availability when at least one station has data. A time slot is considered available if either TMS or TCR (or both) has data and is not in maintenance. This represents overall system coverage.

Note: All times are in HKT for radar data processing. Data processing runs daily at 9:30am for the previous day, so current day data may not be available until the next morning.

How Satellite Availability is Calculated?

What is a Found Data Slot?

A data slot represents a 10-minute time period (00, 10, 20, 30, 40, 50 minutes past each hour), creating 6 slots per hour. However, 02:50 UTC and 14:50 UTC are systematically excluded every day for operational reasons, resulting in 142 available slots per day instead of the theoretical 144. A slot is marked as "found" when the system detects a satellite image file for that specific time period.

Source: Internet > Eastern Asia > IR (Internet/H8_EasternAsia_IR)

Expected Slots Calculation:

Every day at 02:50 UTC and 14:50 UTC, there will be no data for these 2 time slots. Therefore, the expected slots per day is 142 (144 - 2).

e.g. the expected slots of July is 142 × 31 (days) = 4402

Himawari Availability:

Availability = (found data slots that are not in maintenance / expected slots) × 100%. Expected slots = 142 per day × number of days in that month.

Preventive Maintenance Adjustment:

When preventive maintenance occurs, those time slots are subtracted from the expected slots, providing a more accurate availability percentage that excludes planned downtime. However, if preventive maintenance occurs at 02:50 UTC or 14:50 UTC, no subtraction is needed since these slots are already excluded.

Data Restoration:

When a "Data Restoration" record is added, it indicates that data previously thought missing for that period actually exists. The corresponding time slots are added back as found data, overriding any type of maintenance (preventive, corrective, or other). However, the systematically excluded slots at 02:50 UTC and 14:50 UTC are never restored, even if a Data Restoration record covers those times.

Note: All times are in UTC for satellite data processing.
